# AI 现实爽片完整工作流规划

推荐把完整生产流程拆成三个职责清晰的 Skill，而不是把搜索、写剧本和生成故事板全部塞进一个超大 Skill。

## 推荐流程

```text
实时热点检索
  → 热点来源核验与近期选题查重
  → 生成 5—8 张选题卡并评分
  → 用户确认 1—2 个选题
  → 去真实化并写成完整爽片剧本
  → 用户确认人物、对白和结尾
  → 生成人物参考图、分镜帧和故事板
  → 输出豆包 / Seedance 分段提示词
  → 生成、发布、记录数据和复盘
```

## 建议的三个 Skill

### `ai-short-drama-topic-scout`

联网检索最近热点，记录来源和时间，优先直接社交平台、热榜聚合和主流新闻。提炼情绪原型，进行近期选题查重，并按现实感、情绪强度、反击清晰度、视觉表现力、制作难度和合规风险评分。

### `ai-short-drama-script-builder`

把用户选中的选题卡写成完整剧本。默认结构为第一秒冲突、反派嘴硬、主角用规则或证据反击、反派吃瘪、结尾留下评论争议点。删除真实人名、地点、机构和品牌，明确这是虚构剧情演绎。

### `doubao-storyboard-video`

接收锁定后的剧本，负责拆分片段、人物参考图、分镜帧、故事板大图、视频提示词和生成失败后的降敏方案。

## 为什么设置两个确认关口

选题确认可以避免为不值得制作的方向写完整剧本；剧本确认可以避免在对白、人物或反转仍会修改时提前生成大量图片。图片阶段成本最高，应该在文本基本锁定后开始。
